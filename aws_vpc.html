Amazon Web Services

Virtual Private Cloud Setup

There are 6 subnets in this VPC allowing for partitioning of security and high availibilty across two availibility zones, us west 2a, and us west 2b. The reliability of each of these instances is 99.95% uptime per month as guaranteed by Amazon, therefore having parrallel infranstructure in each availibility zone ensures constantly availible services.

This AWS VPC is managed via open vpn with credentials from active directory. Normal clients have access to specific services such as DNS (port 53), and authentication (port 389). This VPC also features two ubuntu instances with an interface in the DMZ and another in the Core. This allows the Domain Controllers to securely connect to the internet.

The server you are on right now is one of the "munki" instances shown in the above diagram of the VPC. These Apache web servers serve this website as well as the munki repository. This is normally secured with client and server side SSL Certificates. For the purpose of this demonstraton, the munki repository can be viewed here.

corp.mattsdemo.ax.lt is the subdomain to mattsdemo.ax.lt and is the internal DNS zone for this demonstration. Incoming requests to mattsdemo.ax.lt are distributed to the web servers using Amazon's Elastic Load Balancer. This also drastically improves the speed at which clients can install updates as well as the total number of clients which can be simultaneously served.
