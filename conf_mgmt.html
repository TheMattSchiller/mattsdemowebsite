Configuration Management

Configuration Management for Mac and Linux

Mac Configuration Managment with Munki, DeployStudio, and AutoPkg

DeployStudio allows for Netboot installation of MacOS Sierra, which is then bound to Active Directory and bootstrapped into Munki. This allows for "Thin Imaging" of client workstations as well as mastering images for rapid deployment. This allows for devolopment for the MacOS platfor via tools like VMware Fusion and VirtualBox, as well as remote client administration for laptops and workstations, keeping applications at the same version across an organization.

Linux Configuration Management with Chef

Amazon Web Services and Chef allow for rapid deployment and configuration management for the Ubuntu 14.04 instances in this demonstration.

The webserver you are on now is one of the five Ubuntu 14.04 instances running in this demonstration. Their configuration is handled by Chef and deployment via Amazon Web Services ami. This allows for consitancy, ease of managment, scalability, and ensures that the enviornment is reproducable.

Expandability to Windows

LDAP is provided via Active Directory and configuration managment via Group Policy. DNS is also handled for vpn clients and within the VPC by Windows.

This system could easily be scaled with a package manager like PDQ Deploy, IMB Big Fix, or HEAT LANRev/Filewave to replicate the Mac Configuration Management
